# 两数之和 — 写给非科班同学的讲解

## 一句话总结

在一堆数字里，找出两个加起来刚好等于目标值的数字，返回它们的位置。

## 心智模型：哈希表就像一本字典

想象你手里有一本字典。你翻到一个词，很快就能找到它的意思——不需要从第一页开始一页一页地翻。

哈希表就是程序里的"字典"：

- **key**（键）= 你见过的数字
- **value**（值）= 这个数字在数组里的位置

查一个词在不在字典里，非常快。哈希表查一个数字在不在里面，也是一样快。

## 算法在做什么？三步循环

```
对于数组里的每一个数字：
  1. 看一眼当前数字（读数组）
  2. 算出"补数"——就是 target 减去当前数字
  3. 翻字典查补数在不在：
     - 在 → 找到答案了！返回补数的位置和当前位置
     - 不在 → 把当前数字和位置记到字典里，继续下一个
```

用更生活化的比喻：你在参加一个派对，每个人身上都写着一个数字。你要找两个人，他们身上的数字加起来刚好是 9。

你一边走一边记住每个人身上的数字和他们站的位置。每遇到一个新人，你就翻一下你的小本子：有没有人身上写的数字，加上我眼前这个人的数字，刚好等于 9？

## 可视化应该展示什么

1. **数组**：一排格子，当前正在看的那一格高亮
2. **哈希表**：旁边一个不断增长的"字典"，每次新增一个条目时动画展示
3. **当前索引 i**：一个指针或者高亮标记，表示"我现在走到哪了"
4. **补数计算**：展示 `target - num` 的过程，让观众看到"我在找什么"
5. **找到答案的瞬间**：两个索引同时高亮，哈希表里的匹配条目也高亮

## 新手常犯的错误

### 1. 忘记"先查后存"

有些同学会先把当前数字存进哈希表，然后再查补数。这样做会导致自己和自己匹配——比如 `[3, 2, 4]` 找 target = 6，你可能会错误地认为 3 + 3 = 6（用了同一个位置两次）。

**正确顺序**：先查补数在不在，找不到再存当前数字。

### 2. 不理解返回的是索引而不是值

题目要的是**位置**（第几个），不是数字本身。比如 `[2, 7, 11, 15]` 找 target = 9，答案是 `[0, 1]`（第 0 个和第 1 个），不是 `[2, 7]`。

### 3. 担心顺序问题

返回 `[0, 1]` 和 `[1, 0]` 都是对的——题目说了顺序不重要。我们的算法天然返回 `[先存的那个位置, 当前位置]`，这没问题。

### 4. 以为哈希表很复杂

哈希表在 Python 里就是字典（`dict`），用法特别简单：

```python
seen = {}          # 创建一个空字典
seen[7] = 1        # 存：数字 7 在位置 1
seen[7]            # 取：得到 1
7 in seen          # 查：True，7 在里面
```

就这么简单。不要被"哈希表"这个名字吓到。

## 为什么用哈希表而不是暴力？

暴力方法是两层循环，每个数字都和其他所有数字配对一次，时间是 O(n²)——数组有 100 个数就要比较将近 10000 次。

哈希表方法只遍历一次数组，每次查表是 O(1)，总共 O(n)——100 个数只需要 100 次操作。

这不是炫技，是真的快很多。

## 小结

| 概念 | 用大白话说 |
|------|-----------|
| 哈希表 | 一本字典，查东西很快 |
| 补数 | target 减去当前数字，就是"我还差多少" |
| 先查后存 | 先翻字典找补数，找不到再把当前数字记下来 |
| 返回索引 | 返回的是位置，不是数字本身 |
