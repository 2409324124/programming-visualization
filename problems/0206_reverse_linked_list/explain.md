# 反转链表 — 通俗解释

## 这道题在做什么？

想象你有一排人，每个人手里拿着一根绳子，绳子的另一头系在下一个人身上。
这根绳子就是"箭头"（next 指针）。

**反转链表 = 把所有绳子的方向反过来。**

原来 1 → 2 → 3 → 4 → 5，反转后变成 5 → 4 → 3 → 2 → 1。

---

## 箭头模型：三个关键角色

我们用三个变量来完成这个任务：

| 变量 | 比喻 | 作用 |
|------|------|------|
| `prev` | 已经修好箭头的那组人 | 指向"已经反转好的部分"的头 |
| `curr` | 正在被修箭头的那个人 | 指向"正在处理的节点" |
| `next_temp` | 暂时记住后面还有谁 | 在断开箭头之前，先把"下一个人"存起来 |

---

## 四步循环（每一轮都做同样的事）

假设当前处理的是节点 3（curr = 3）：

```
第一步：存 next
    next_temp = curr.next    // 记住 4 在哪里，不然等会儿断了箭头就找不到了

第二步：断箭头、反方向
    curr.next = prev         // 把 3→4 的箭头改成 3→2

第三步：prev 前进一步
    prev = curr              // prev 来到 3 的位置

第四步：curr 前进一步
    curr = next_temp         // curr 来到 4 的位置（之前保存的）
```

然后下一轮，处理节点 4……直到 curr 变成 None，说明所有节点都处理完了。

---

## 常见误区

### 误区 1：忘了保存 next_temp

```python
# ❌ 错误写法
curr.next = prev   # 断开箭头后，curr.next 已经不是原来的下一个了
prev = curr
curr = curr.next   # 这里 curr.next 已经变成 prev 了！链表断了！
```

**记住：必须在断箭头之前，先把"下一个人"存到 next_temp 里。**

### 误区 2：不理解为什么 prev 是新的头

循环结束时，`curr` 是 `None`（走到了链表末尾的后面），而 `prev` 刚好停在最后一个节点上——也就是反转后的新头部。

---

## 如何看 Trace（执行轨迹）

每一步 trace 会显示：

- **节点的值**：当前 prev、curr、next_temp 分别指向哪个节点
- **箭头方向**：哪个节点的 next 指向了谁
- **通俗解释**：这一步在做什么，为什么要这样做

关注这些变化，你就能"看到"链表是怎么一步步被反转的。

---

## 一句话总结

> 反转链表 = 一边往前走，一边把路过的箭头全部掉头。走到头的时候，最后一个节点就是新的起点。
