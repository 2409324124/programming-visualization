# 打家劫舍 — 写给非科班同学的讲解

## 一句话总结

你面对一排房屋，每间房都有一笔金额。你不能选择相邻的两间房，问最多能拿到多少钱。

## 心智模型：每间房都做一次选择

走到第 `i` 间房时，你只有两个合法选择：

1. **跳过当前房**：那最大金额还是 `dp[i-1]`
2. **偷当前房**：因为不能偷相邻房，所以只能拿 `dp[i-2] + nums[i]`

所以：

```text
dp[i] = max(dp[i-1], dp[i-2] + nums[i])
```

`dp[i]` 的意思不是“第 i 间房一定偷了”，而是：

> 看完第 i 间房之后，在合法规则下最多能拿多少钱。

## 为什么不是看到大数字就拿？

局部看起来大的房子，可能会挡住后面更好的组合。DP 的作用就是把“偷当前房”和“跳过当前房”两条路线都算清楚，再保留更大的那个。

## 算法步骤

```text
1. 如果没有房屋，返回 0
2. 创建 dp 表格，长度和 nums 一样
3. 从左到右处理每间房：
   - i=0：只能选择偷或不偷第一间
   - i=1：只能在第 0 间和第 1 间中选更大的
   - i>=2：比较 dp[i-1] 和 dp[i-2] + nums[i]
4. 返回最后一格 dp[-1]
```

## 可视化应该展示什么

1. 房屋金额数组：每间房的金额
2. DP 表格：每个位置保存“到这里为止的最大金额”
3. 读取的旧格子：`dp[i-1]` 和 `dp[i-2]`
4. 当前选择：跳过当前房 vs 偷当前房
5. 写入结果：把更大的金额写进 `dp[i]`

## 新手常犯的错误

### 1. 把金额最大的房子一定选上

最大单间房不一定属于最优组合。你要比较的是整条路线的总金额。

### 2. 忘记相邻限制

如果偷第 `i` 间房，就不能偷第 `i-1` 间。因此偷当前房时只能接 `dp[i-2]`。

### 3. 误解 `dp[i]`

`dp[i]` 不是“第 i 间房偷到的钱”，而是“看完第 i 间房后的最好总金额”。

## 小结

| 概念 | 用大白话说 |
|------|-----------|
| `dp[i]` | 看完第 i 间房后最多能拿多少钱 |
| 跳过当前房 | 沿用 `dp[i-1]` |
| 偷当前房 | 当前金额 `nums[i]` 加上安全位置 `dp[i-2]` |
| 最终答案 | 最后一格 `dp[-1]` |
